# codegraph 0.6.0 — "Trust" (design + implementation plan)

Theme: close the three gaps that force agents and humans to hedge — ambiguous
nav silence, token-match testRefs, oversold README — and delete the artifact
weight that no longer earns its churn. Everything here is a precision or
honesty fix; the two later releases build on it.

Standing doctrine: see plans/ROADMAP.md — applies verbatim.

## Wire-format additions (pinned; all additive)

- `stats` gains a FIRST key `format: 3` (int, schema generation). Query-side
  `Graph.load()` warns on stderr (one line, never fails) when `format` is
  missing or greater than the binary's known version — the cross-machine
  stale-binary tell. (Missing = pre-0.6 graph: fine, just note it.)
- `imports` edges gain optional final key `line` (int, 1-based line of the
  import/export/part directive; first occurrence wins for duplicate imports
  of the same target). Needed by 0.7.0 lint for file:line violation output.
- `navigates` edges gain optional final key `unresolved: true`, present
  exactly when NO sibling `navigates-to` edge with the same `src` and `line`
  was emitted. (Resolved navs stay unmarked — presence of the sibling is the
  positive signal; the flag makes the negative legible without set math.)

## Stage 1 — nav legibility: gaps must be visible, not silent

Problem: `wiring`/`brief` show `navigates-to` only when resolution succeeded;
an agent cannot distinguish "resolver blind spot" from "page navigates
nowhere". 39% resolution with invisible gaps erodes trust in the whole graph.

- engine: emit the `unresolved: true` flag per the pin above (computed at
  edge-emission time from the same-src-same-line sibling check — build-time
  data only, deterministic).
- query `wiring` + brief file cards: render the navigates section with the
  resolution inline:
  - resolved:   `'/details':12 → lib/x/details_page.dart`
    (target = the sibling navigates-to dst; join on src+line)
  - unresolved: `AppPaths.foo.path:34 (unresolved)`
  Drop the separate `navigates-to (N):` section from wiring/brief (it becomes
  redundant once targets render inline) — `path` traversal keeps using the
  edges as today. This is a RENDER change to two verbs; `--json` for wiring
  keeps both arrays as-is PLUS adds `unresolved` on navigates records.
- passport: append resolution coverage to the header area, one clause:
  `nav: 39/100 resolved` (numbers from counting edges at load — cheap).
- Tests: fixture asserts the flag placement (present on the local-var caller,
  absent on the resolved caller), the new inline render for both cases, and
  that `path` behavior is unchanged.

## Stage 2 — slim the markdown maps to Summary-only; delete the splitting machinery

Decision (recorded here so it isn't re-litigated): the per-area `.md` files
predate `brief`. Agents should orient via `brief <area>` (fresh, budgeted,
cheaper); the maps' only remaining audience is humans browsing GitHub. The
wiring tables and file inventories duplicate what the verbs answer better,
and they are the bulk of the commit churn on every code change.

- `_renderMarkdown` emits ONLY: title, the existing `## Summary` section
  (counts, providers by reader count, entry pages, cross-area providers),
  and a one-line pointer: `Full detail: codegraph brief <area> · wiring ·
  sym · skeleton`. Delete the "Providers declared here", "Provider wiring"
  table, "Navigation", and "File inventory" sections.
- DELETE the oversized-map splitting machinery entirely — `splitDecision`,
  `_writeSplitAreaMap`, the 20k-char threshold, sub-map INDEX handling — a
  Summary-only map cannot exceed the threshold. Also delete their tests and
  the sub-map link rendering. (Hosts will have stale `<area>-<sub>.md` files
  on disk; `build` doesn't delete unknown files — the host rollout step
  removes them once, documented in the release notes.)
- INDEX.md: keep, recompute `~tokens` against the new sizes.
- This is the release's one behavior-visible artifact change: hosts see a
  one-time large diff in committed maps (shrinkage). CHANGELOG must state the
  rationale (brief supersedes; churn per PR drops ~10x) prominently.
- While in the writer: EXTRACT the whole markdown/INDEX/ATTENTION rendering
  block (~360 lines) from engine.dart into lib/src/markdown.dart (pure
  refactor riding along; nav_resolution.dart established the pattern).
  engine.dart should land near ~1,100 lines.
- Tests: summary-only golden-ish assertions; determinism lock unchanged;
  engine emits identical code_graph.json (maps change, graph must NOT —
  assert the fixture graph bytes match a pre-stage snapshot).

## Stage 3 — testRefs precision: registry resolution replaces bare tokens

Problem: a provider name in a comment in ANY test file counts as a test
reference. `untested` is therefore "candidate data" and 0.8.0's
affected-tests cannot be trusted to SKIP tests on it.

- In `_scanTestRefs`, a provider-name token in test file T now counts ONLY
  when at least one declaration of that provider is in T's import+export
  closure (the closure is already computed for file credits — reuse it).
  Same per-test-file dedup. Ambiguous names: credit every declaration whose
  file is in T's closure (unchanged behavior, now closure-gated).
- Same gate for any future token-based crediting; document the residual
  honestly (a comment naming an IMPORTED provider still counts — a far
  smaller class; grep hedge shrinks accordingly in README/untested output).
- The `· test-only (N test refs)` suffix and `untested` need no code change —
  their inputs get more precise.
- Expect reference host numbers to shift (some token-only credits vanish). Stage
  gate: run `untested` on reference host before/after; spot-check 3 providers that
  changed classification by opening their tests — each reclassification must
  be correct. Report the before/after counts in the stage report.
- Tests: a test file mentioning a provider it does NOT import → no credit;
  mentioning one it imports via a barrel → credit (existing barrel fixtures
  extend naturally).

## Stage 4 — doctor, monorepo self-location, README honesty

- NEW verb `codegraph doctor` (read-only, prints findings + the fix command
  for each, exit 1 if any check fails, `--json`):
  1. binary version vs graph `format` (stale binary / stale graph tell);
  2. graph exists / builds (`docs/maps/code_graph.json` present);
  3. hook file present + executable + wired in `.claude/settings.json`
     (valid JSON, contains the hook path);
  4. `.gitignore` has the code_graph.json line; the JSON is not git-tracked;
  5. CLAUDE.md marker block present; skill file present;
  6. CI workflow present (note-level, not failure, when absent);
  7. package root == git root, else print the monorepo guidance (below).
  This kills the "silently broken install" class the escaping bug and the
  a nested-package monorepo already demonstrated. init's "Next steps" mentions
  `codegraph doctor`.
- Monorepo: the generated hook self-locates — if
  `$CLAUDE_PROJECT_DIR/pubspec.yaml` is missing, probe one directory level
  for a subdir containing both `pubspec.yaml` and `docs/maps/` and cd there
  (first match by sorted name; none found → exit 0 silently, fail-safe as
  ever). `init` run from a package root that is not the git root prints the
  root-level-settings guidance instead of leaving it to a CHANGELOG footnote.
- README: replace "works on any project unchanged" framing with the honest
  positioning: deep value on Riverpod + GoRouter (wiring, nav, providers);
  imports/types/symbols/tests everywhere else. One short paragraph.
- Consolidate the ponytail-audit leftovers while touching these files: one
  `Budget` class, one `joinCapped`, one `resolveFileArg`, one `inDegSuffix`,
  one `bare` in cli_util.dart; delete the four copies (net ~-52 lines; pure
  moves, behavior identical).
- Tests: doctor happy path + one broken check per class (missing hook,
  tracked JSON) in a fixture; hook self-location exercised in a two-level
  fixture via the subprocess harness.
