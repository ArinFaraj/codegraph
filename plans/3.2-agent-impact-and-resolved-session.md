# 3.2 — prove agent impact, then remove resolved-analysis friction

## Why this is next

Production-scale Flutter dogfooding proves both halves of the v3 thesis:

- cached graph queries are already interactive (roughly 60–130 ms), and
- element-precise rename can outperform text search on a real interface
  hierarchy by selecting 15 semantic edit sites while excluding two
  same-spelling strings.

It also exposes the current adoption blocker. Resolved builds and independent
resolved operations take tens of seconds with no visible progress. Accuracy is
good enough to evaluate; the remaining question is whether it improves an
agent's end-to-end work, and then how cheaply the analyzer context can be
reused.

Run two otherwise identical capable-agent arms in disposable,
production-shaped Flutter benchmark workspaces:

1. **Baseline:** `rg`, file reads, Dart Analysis Server/LSP rename,
   `dart analyze`, and tests.
2. **Treatment:** every baseline tool plus codegraph.

Use the same model, prompt, time limit, token limit, starting commit, and task
oracle. Alternate arm order and run each task three times.

### Task set

1. Rename a standalone public function.
2. Rename a private helper with unrelated same-name symbols nearby.
3. Rename a Riverpod provider/Notifier pair.
4. Rename a method across an interface, production implementations, and test
   fakes.
5. Refuse an unrelated same-name collision.
6. Refuse a framework override such as `Object.noSuchMethod`.
7. Refuse a cross-package/public-boundary edit whose completeness is unknown.
8. Refuse an unsupported signature change.

### Oracle and gates

The oracle records the exact required edit set or the exact required refusal.
After every attempt, run formatting, analysis, targeted tests, and the full
test suite.

Primary measures:

- task success;
- exact edit-set completeness;
- zero unrelated edits;
- correct refusal;
- analysis and tests stay green.

Secondary measures:

- elapsed time;
- tool calls and tokens;
- recovery loops after a failed attempt.

Do not expand actuator scope unless the treatment has 100% safety and either:

- improves task success by at least 20 percentage points, or
- reduces time or tool use by at least 30% without reducing task success.

Commit the benchmark runner, task fixtures, immutable oracles, and raw results
so future releases can reproduce the claim.

## Stage B — make long operations observable

Before optimizing, add stable progress events to resolved build, callers, and
rename. Stderr-only, rate-limited file progress is now implemented. Rename
apply also prevalidates all spans and uses staged, rollback-backed installation;
the remaining work is richer phases and explicit cancellation:

- phase name (`discover`, `resolve`, `select`, `validate`, `render`);
- completed and total file counts when known;
- elapsed time;
- a cancellation path that leaves no partial committed artifact.

Human output may redraw a single status line. JSON mode emits newline-delimited
events only when explicitly requested so existing machine output remains
stable. Add timeout and cancellation tests.

## Stage C — persist reusable resolved identities

Status: rename, resolved callers, and resolved refs implemented.

Resolved build now emits a deterministic, gitignored refactor index with stable
executable identities, declaration/body spans, reference sites with enclosing
caller identity, and override links across production and test files. Rename
and tracked-body affected-test attribution consume it only when the index is
complete and its source digest matches the graph. Otherwise they fall back to
file-level or a fresh analyzer context. The artifact also records unbound identifier spellings, so a
dynamic or unresolved use of the target forces refusal instead of disappearing
from the edit set. A syntax build removes the index.

This was chosen over a resident worker because it reuses the resolved work
without adding a background process, workspace isolation rules, idle shutdown,
or a second state model. Existing CLI contracts stay unchanged; affected-test
JSON adds `changeAttribution`, `affectedSymbols`, and `precisionFallbacks`
evidence.

Call-vs-reference kind and unresolved target sites now preserve the existing
grouped-target output for resolved callers/refs, with parity tests against the
cold analyzer path. Reconsider a resident language-server-style worker only if
a future semantic operation cannot be represented safely as a deterministic
artifact.

Resolved typed GoRouter receivers now add a complementary contract edge: route
data `.go/.push/.replace/.pushReplacement` calls connect to pages returned by
exact direct or single-return `build`/`buildPage` bodies. The affected-test
oracle freezes this reverse dependency. Annotation trees, shells, redirects,
and navigator ownership remain explicit follow-up work rather than being
inferred from names.

Research basis:

- Dart's SDK has moved command-line analysis onto its AOT analysis server and
  continues to improve analyzer/plugin performance.
- LSP and the Dart Analysis Server are excellent interactive baselines, with
  persistent workspace state, references, rename, cancellation, and progress.
- `AnalysisContextCollection` supports a reusable resolved context but requires
  explicit disposal; the build and cold fallback now honor that lifecycle.

The product conclusion is not to duplicate an IDE protocol. Codegraph's
advantage is a compact, deterministic, agent-oriented artifact with strict
refusal gates; the analysis server remains the honest baseline.

## Stage D — turn proven invariants into conformance

Mine real failures from the impact benchmark and host repositories. Implement
only high-signal layering, pairing, and must-stay-in-sync rules with fixtures
for violation, non-violation, and uncertainty/refusal.

The CLI remains the source of truth. Once the repo can require Dart 3.10+, an
optional analyzer-plugin adapter may surface the same diagnostics and quick
fixes in IDEs; it must not fork rule logic.

## Smaller operational follow-ups

- Define and test the supported distribution shape. A standalone native

  do not advertise it until that path is proven.
- Keep graph analysis mode and resolved coverage prominent in every trust- or
  edit-sensitive response.

## Exit criteria

3.2 is complete when the public benchmark is reproducible, indexed rename is a
regression-gated speedup, resolved callers/refs use the same artifact, and
write application is prevalidated, rollback-backed, and cancellable.
