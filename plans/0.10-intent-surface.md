# codegraph 0.10 -> 0.12 - "Intent" (verb-surface redesign plan)

Theme: the engine is ~90% of its potential; the SURFACE is ~40%. Extraction is
accurate (a 2026-07-10 hand-verified audit on the reference host found zero
false edges across ~120 classified reads/calls/subtypes), but the tool exposes
22 verbs that mirror graph internals instead of the handful of questions an
agent actually asks. Every point of confusion logged in the reference host's
LIMITATIONS.md is a surface failure, not an engine failure. This plan collapses
the surface around agent intents, makes staleness impossible, and types every
empty answer - then proves the redesign with the benchmark instead of vibes.

Standing doctrine: see plans/ROADMAP.md - applies verbatim. Especially:
never-guess (refusals ship with tests), line-budgeted output, additive wire
format, deterministic artifacts.

## Why the surface is the bottleneck (read before re-litigating)

1. **Verb choice is a zero-context decision.** The agent picks one of 22 verbs
   BEFORE any information arrives. Overlapping names (`readers` / `provider` /
   `wiring` / `refs` / `callers`; `find` / `sym` / `brief` / `skeleton`) mean
   the pick is often wrong, and a wrong pick is an invisible failure: `find`
   returns nothing where `sym` would hit; `readers <Class>` needs a redirect
   message. The 0.9.3 "not a provider - use find/wiring" fix was a patch on
   this design bug: the graph's internal taxonomy should never leak to the
   caller at all.
2. **Round trips are the expensive unit for an agent, not output lines.** A
   second tool call costs latency, reasoning tokens, and a chance to give up
   or hallucinate. The usefulness benchmark literally scores `toolCalls`;
   merged verbs move the metric the benchmark says matters most.
3. **Silent partial answers are the actual "AI confusion".** The audited
   failure modes were: stale graph indistinguishable from absence, lib-only /
   file-level scope surprises, candidate lists read as verdicts. None were
   wrong edges. Fixing the envelope (freshness, typed empties, inline caveats)
   removes the confusion class without touching extraction.
4. **Standing token cost.** The verb list is injected into every session via
   skill/CLAUDE block. 22 verbs of documentation is a per-session tax; 5
   intent verbs shrink the scaffolding and the model's decision space.
5. **LLMs are trained on git-shaped tools**: few verbs, polymorphic rich
   output, `status` telling you what to run next. That is the shape to copy.

## The intent model

Agents ask five questions. Every current verb is a partial answer to one:

| Agent intent | Today's verbs (22) | Target verb (5 + plan) |
|---|---|---|
| What/where is X? | find, sym, skeleton, brief, provider (decl half), passport | **find** |
| Who uses / depends on X? | readers, callers, refs, wiring, provider (consumers half), impls, callchain, path | **uses** |
| What must change / breaks if I touch X? | impact, impls (again), untested (per-thing), the shape-change hint | **change** |
| Is my branch safe? | diff, lint (report), untested (diff scope) | **review** |
| Where should I start / what is unhealthy? | attention, unused, untested (repo scope), blueprint | **health** (+ keep **plan**/blueprint) |

Operator verbs stay as-is and out of the agent's way: `build`, `check`,
`init`, `upgrade`, `doctor`, `lint` (CI gate mode). `passport` becomes
hook-only (hidden from help).

## Target surface (agent-facing)

### `codegraph find <anything>`  - the identity front door
Absorbs find + sym + skeleton + brief + provider's declaration half.
ONE resolver (token + phrase + case-insensitive + member names + file paths),
then a card whose shape follows what the query resolved to:
- symbol/member -> signature, doc, declaration(s) file:line, members, and
  COUNTS of every inbound relation with the follow-up spelled out
  ("14 call sites -> `codegraph uses X`").
- file -> skeleton (declarations + lines) + imports/importedBy counts.
- provider -> declaration, kind, autoDispose, reader counts.
- directory/area -> brief-style area card (notes sidecar included).
- several candidates -> ranked list (in-degree), refuse to pick.
Resolution semantics become UNIFORM for every verb because they all share this
resolver - today each verb has its own matching quirks, which is why agents
learn distrust.

### `codegraph uses <thing>`  - every inbound edge, sectioned
Absorbs readers + callers + refs + wiring(inbound) + provider's consumers half
+ impls + callchain (as `--chain`).
Sections auto-selected by node kind: provider -> watch/read/listen (+ per-
declaration narrowing, cascades, container reads); method/function -> call
sites, then tear-offs/refs separately; type -> transitive subtype tree incl.
test fakes, then type-uses; file -> importers + navigates-here. The agent
never chooses between readers/callers/refs again, and "X is a class, not a
provider" dies as a message because the question no longer has a wrong door.
`--chain` adds the callchain tree with its hazard flags (unique value, keep).

### `codegraph change <thing>`  - the pre-change pack
Absorbs impact + the shape-change hint + per-thing untested, and composes the
one answer agents actually need before an edit: direct + transitive dependents
(depth-tiered), subtype tree incl. test fakes, state-type users for providers,
test coverage of the touched set, ranked by in-degree. The canonical agent
failure ("renamed the provider, missed the Notifier subclasses") becomes
impossible to reach via a partial verb because there is no partial verb.

### `codegraph review [--base main]`  - branch safety card
diff + lint findings scoped to changed files + changed-but-untested. Already
90% built as `diff`; this is a rename plus lint absorption. CI keeps posting it.

### `codegraph health`  - repo triage
attention + unused + untested merged, with the candidate-list framing printed
INLINE per section ("candidates - confirm with exact-path grep across all
Dart roots + flutter analyze before deleting"), not buried in a doc.

### `codegraph plan <feature-dir>`  - blueprint, renamed for intent symmetry.

## Deleted / demoted (record here so it is not re-proposed)

- `path A B` - DELETE. Documented as unintuitive (routes through shared
  utilities), zero dogfood wins logged since 0.2. If a real need reappears,
  it re-enters as `uses --route` with import-weighting, not as a verb.
- `provider`, `wiring`, `refs`, `readers`, `callers`, `sym`, `skeleton`,
  `brief`, `impls`, `impact`, `attention`, `unused`, `untested`, `callchain`,
  `blueprint` - become hidden ALIASES for one minor release (0.11 -> 0.12),
  then are removed from help; aliases keep working indefinitely (they cost
  nothing and old transcripts/templates reference them).
- `passport` - hook-only, hidden from help.

## Cross-cutting mechanics (this is where trust is won)

1. **Staleness becomes impossible.** `build` records a source-content hash
   (file list + mtimes/sizes digest) in the UNTRACKED code_graph.json. Every
   query verb re-derives the digest (cheap stat pass over lib/ + packages/),
   and on mismatch AUTO-REBUILDS (build is ~2s on a 1.5k-file host) with one
   stderr line: `graph was stale (N files changed) - rebuilt`. Deterministic,
   no wall-clock in artifacts, kills the #1 documented confusion (silent
   false negatives from a stale graph). `--no-rebuild` opts out for CI.
2. **Typed empties.** "(no matches)" never appears alone. Three distinct
   shapes, both text and JSON:
   - `not found (graph fresh, N files indexed; nearest tokens: ...)`
   - `outside the graph's scope (runtime/behavioral question) - read source`
     (emitted when the query resolves but the asked relation is one the graph
     does not model)
   - `ambiguous (K declarations) - refusing to pick` + the list.
3. **Caveats travel with answers, not docs.** Every card ends with one
   caveat line derived from the verb + node kind ("file-level; lib-only;
   wrapper-object refs not detected"), and `--json` carries `caveats: []`
   machine-readable. LIMITATIONS.md stays the long-form registry; the answer
   carries the one line that prevents over-trust at the moment it matters.
4. **One JSON envelope, pinned** (additive wire change):
   `{ verb, query, resolvedAs, fresh, results, truncated, caveats }` on every
   query verb. No more per-verb JSON availability roulette.
5. **Counts-first budgets.** Cards lead with counts + follow-up commands; full
   lists render within `--budget` as today, `--full` lifts the cap. An agent
   gets the shape in 20 lines and pays for detail only when it asks.
6. **Exit codes**: 0 = answered (including a typed empty), 2 = could not
   answer (ambiguous / unbuildable) - so scripts and hooks can branch without
   parsing prose.

## Proof gates - the redesign must EARN the break

This is the part that keeps 0.10-0.12 honest. Before any collapse ships:

1. Add verb-choice micro-scenarios to the reference host's agent A/B harness:
   questions whose correct answer today requires picking the right verb among
   near-synonyms. Measure wasted round trips per answer on the CURRENT
   surface (baseline).
2. Gate each stage on the deterministic suites (fixture floors + real-repo
   invariants) staying green, PLUS: agent-arm toolCalls per scenario goes
   DOWN, recall/completeness stays flat, per-answer outputChars stays within
   budget, and the A/B overall does not regress.
3. `upgrade` rewrites host scaffolding (skill, CLAUDE block, hook) to the new
   verb list in the same release that ships the new verbs - templates and
   binary never disagree.

## Staging

- **0.10 "Trust the envelope"** (no surface break): shared resolver behind
  the existing verbs, freshness hash + auto-rebuild, typed empties, caveat
  lines, JSON envelope + exit codes. Every existing verb gets more honest;
  nothing an agent memorized breaks. Acceptance: both deterministic suites
  green; new fixture tests for each typed-empty shape and the staleness
  rebuild; byte-stable committed artifacts.
- **0.11 "Intent verbs, additive"**: ship `uses`, `change`, `review`,
  `health`, `plan` ALONGSIDE the old verbs (old = aliases). Run the A/B
  verb-choice scenarios old-vs-new prompts; record the deltas in the host
  bench README. Acceptance: proof gates above.
- **0.12 "Collapse"**: `find` absorbs sym/skeleton/brief; help text fits one
  screen (5 agent verbs + 6 operator verbs); aliases hidden; scaffolding
  templates rewritten via `upgrade`. Acceptance: help under 30 lines; a fresh
  agent session on the reference host completes the micro-scenario battery
  with fewer tool calls than the 0.9.8 baseline.

## Rejected for this redesign (do not re-propose without a new trigger)

- Natural-language query verb (LLM inside the CLI) - breaks determinism, the
  tool's entire edge.
- Auto-picking a declaration on ambiguity to "help" - violates never-guess.
- MCP server mode - already rejected (resident schema token cost); the
  intent-verb surface makes the CLI cheaper than a server could ever be.
- Embedding/BM25 search - the shared resolver's token+phrase+member matching
  covers the observed misses; revisit only with a logged failure the resolver
  cannot express.
