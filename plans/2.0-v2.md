# codegraph 2.0 - "v2" (the everything-the-audit-found release)

Mandate: a 2026-07-11 four-lens adversarial audit (architecture, positioning,
correctness ceilings, measured performance/ops) found the engine precise but
the project at ~40% of its potential. 2.0 fixes every finding that is fixable
without abandoning the current execution model, ships the intent surface, and
bumps the wire format. 2.1 (separate plan) is the resolved-analysis daemon -
the one change that needs its own foundation and amends the syntax-only
doctrine with explicit conditions instead of "Ever."

Standing doctrine still applies (plans/ROADMAP.md) with ONE amendment staged
for 2.1: doctrine item 1 (syntax-only, no pub get) gains a reopening clause.

## Batches (each lands green before the next starts)

### A - mechanical hygiene
- `providerConsumerRels` const in model.dart replaces 7 literal copies of
  {'watches','reads','listens'}; `untestedRoles` deduped (query + diff).
- Shadow copies deleted: skeleton's _intFlag/_emit, brief's emit,
  attention's _intFlag all use cli_util's single definitions.
- One `runGit` guard helper replaces 6 hand-rolled ProcessException guards.
- test/version_lock_test.dart: binaryVersion must equal pubspec version
  (the two already drifted once; this is the tripwire).
- init templates pin activation to `--git-ref v<version>` - hosts stop
  silently tracking main.

### B - correctness
- callchain: ambiguity refusal made unconditional (was skipped exactly at the
  depth cap, silently attributing decls.first's file/line/hazards - a direct
  doctrine violation). Regression fixture: two same-named methods on
  unrelated classes.
- Mixin `on` constraints and extension-type `implements` captured as subtype
  edges (stated facts, previously dropped silently by _collect).
- Parse diagnostics surfaced: build prints one stderr line when N files had
  parse errors (previously a half-edited file folded silently into "truth").

### C - structure
- ONE shared argument resolver (typed Resolved/Ambiguous/NotFound result)
  behind all six divergent per-verb resolvers (path/wiring/skeleton/impact/
  brief/blueprint had different match and ambiguity semantics for the same
  input). Ambiguous refusal exits 2 (the cannot-answer code); 0 stays
  "answered", including typed empties.
- FileInfo + registry types move out of engine.dart to their own module;
  the engine<->nav_resolution import cycle is broken for real (the comment
  claiming one-directionality becomes true).
- engine's module globals (_self/_packages) become parameters.
- GraphEdge gains typed childName/parentName; the `detail` string
  ' -> ' re-parsing dies. WIRE FORMAT 2 (v2 may break format; graphFormat
  version bumped, loader warns on old graphs and rebuild fixes them).
- LintConfig.load returns errors instead of exit() (exit stays in run()).

### D - surface + disclosures + perf fast path
- Intent verbs land per plans/0.10-intent-surface.md: `uses` (all inbound
  edges by node kind), `change` (pre-change pack: dependents + subtype tree +
  state-type users + test coverage), `review` (diff + lint on changed files),
  `health` (attention + unused + untested), `plan` (blueprint alias). Old
  verbs remain as aliases; help fits one screen.
- Disclosures added everywhere they matter: ProviderScope overrides are not
  modeled (readers caveat + LIMITATIONS seed), callers/refs name-collision
  count inflation (caveat + per-hit note), provider family collapse,
  testRefs cross-declaration credit bleed.
- Freshness digest fast path: stat walk (paths+sizes+mtimes) first; content
  hash only when the stat digest differs from the stored one. Cuts the
  ~150ms per-query tax to ~10ms steady state without weakening the
  never-stale guarantee (stat mismatch just triggers the content check;
  both digests stored in stats).
- Version 2.0.0; CHANGELOG entry; README rewrite for the new surface.

## Gates (every batch)
dart analyze --fatal-infos, dart format, dart test, perf compare,
usefulness --check (floors updated only for deliberate fixture truth changes,
stated in the commit), and the reference host suite
(tools/codegraph_bench/run.dart) before release.

## Explicitly OUT of 2.0 (goes to 2.1, see plans/2.1-resolved.md)
- Resolved-analysis backend (element-identity callers/readers, type-aware
  receiver detection) and the warm per-session daemon. Doing this hastily
  inside 2.0 would ship a broken foundation; it gets its own staged plan,
  including the doctrine amendment and the fixture/package-config work
  resolution needs.
- Incremental build (belongs to the daemon, not the one-shot CLI).
