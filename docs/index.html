The problem

Grep finds text. It does not know what is related.

"What breaks if I change this?" is a whole-program question, not a text search. A blind agent can read files one by one. The graph knows the whole program.

The 3.0.0 release

Resolved knowledge, then an actuator.

v3 changes the default build to resolved analysis, makes the NEVER-GUESS doctrine visible as data, and adds the first actuator: a rename that refuses before it can break anything.

  1. 01

    Resolved analysis is the default build

    codegraph build now uses the Dart analyzer's element model. Syntax-only is the automatic zero-setup fallback when .dart_tool/package_config.json is missing, and can be forced with --syntax. If you explicitly ask for --resolved and the package config is missing, it refuses with a pub get instruction instead of silently degrading.

    • default is resolved
    • syntax-only is the zero-setup fallback
    • explicit --resolved without package_config refuses
    $ codegraph build
Indexing 1546 files... done in 34s
resolved: 1542/1546 files (99.7%)
4 files fell back to syntax (package_config missing)
graph written to docs/maps/code_graph.json
  2. 02

    Element-precise readers and callers --resolved

    readers and callers|refs <Symbol> --resolved attribute every call site to its real target. HomeScreen.build and SettingsScreen.build are separate result sets, not a same-name union.

    • receiver type, not just name, decides reader edges
    • same-named methods in different classes stay split
    • subtypes and implements edges carry element-confirmed identity
    $ codegraph callers HomeScreen.build --resolved
HomeScreen.build - 2 call sites
  lib/main.dart:22
  lib/router.dart:18
override chain: HomeScreen.build -> StatelessWidget.build -> Widget

$ codegraph callers SettingsScreen.build --resolved
SettingsScreen.build - 3 call sites
  lib/settings/view/settings_screen.dart:44
  lib/profile/view/profile_section.dart:12
  test/settings/settings_screen_test.dart:29
override chain: SettingsScreen.build -> StatelessWidget.build -> Widget
  3. 03

    Confidence column: resolved, heuristic, guessed

    Every GraphEdge carries a confidence value. It is emitted only when it is not heuristic. readers marks [unconfirmed] when a reader is name-matched but not element-confirmed. The build prints an honesty metric so you know the resolved fraction.

    • resolved: element-confirmed
    • heuristic: name/reachability only, normal in syntax builds
    • guessed: never emitted; the tool refuses instead
    $ codegraph readers authRepositoryProvider
provider authRepositoryProvider - declared in lib/auth/data/auth_repository.dart
watches (3):
  lib/auth/view/login_page.dart  [resolved]
  lib/auth/view/signup_page.dart  [resolved]
  lib/auth/view/account_page.dart  [unconfirmed]
  4. 04

    Inheritance and override safety

    callers --resolved reports the target's inheritance override chain. If the chain reaches an external or framework base, the tool flags the method as unsafe to touch. The rename actuator refuses before it can break a framework contract.

    • override chain shown with every resolved call
    • framework/base override = unsafe to rename
    • refusal is the default, not a warning
    $ codegraph callers HomeScreen.build --resolved
HomeScreen.build - 2 call sites
  lib/main.dart:22
  lib/router.dart:18
override chain: HomeScreen.build -> StatelessWidget.build -> Widget
framework base: unsafe to rename

$ codegraph rename HomeScreen.build _buildHome
Refusing: override chain reaches external/framework base StatelessWidget.build
reason: unsafe (cannot rename a framework override contract)
exit code 3
  5. 05

    The rename actuator

    codegraph rename <Symbol|Class.method> <newName> is the first write actuator. It renames the declaration and every element-resolved reference, including the whole in-project override set, together. It refuses if the target is ambiguous, unsafe, incomplete, or missing package_config. Dry-run by default; --apply writes.

    • dry-run shows every file that would change
    • --apply writes the resolved edit set
    • refuses with exit code 3 when it cannot be safe
    $ codegraph rename SettingsController.logout handleLogout
dry-run: 6 changes in 4 files
  lib/settings/settings_controller.dart:31  declaration
  lib/settings/settings_controller.dart:56  self-call
  lib/home/home_screen.dart:88  call
  lib/profile/profile_screen.dart:102  call
  lib/router.dart:19  tear-off
  test/settings_controller_test.dart:41  call
override set: SettingsController.logout (no framework base, safe)

$ codegraph rename SettingsController.logout handleLogout --apply
applied 6 changes in 4 files
Query surface

Every question an agent asks, with the right precision.

Five intent verbs cover the common questions in one call. Low-level verbs give precise control. The rename actuator is the first write verb.

  • codegraph findLocate files, providers, symbols, and members by name.
  • codegraph usesEvery inbound relation, sections picked by what X resolves to.
  • codegraph changeDependents + subtype tree + state follow-ups + untested in blast radius.
  • codegraph reviewBranch safety: blast radius, untested code, lint violations.
  • codegraph healthTriage, dead-code candidates, and coverage gaps.
  • codegraph planBuild-order plan from a feature exemplar.
  • codegraph callersEvery call site of a method or function.
  • codegraph refsReferences to a symbol.
  • codegraph callchainStatic call tree with control-flow hazards.
  • codegraph readersEvery watcher, reader, and listener of a provider.
  • codegraph implsFull subtype tree of an interface, including test fakes.
  • codegraph wiringA file's dependencies in both directions.
  • codegraph pathShortest relationship path between two files.
  • codegraph symSignature, doc comment, members, and importers.
  • codegraph skeletonA file's declarations with line numbers.
  • codegraph unusedDead-code candidates to confirm, not a delete list.
  • codegraph untestedCoverage gaps ranked by impact.
  • codegraph impactTransitive dependents of a provider or file.
  • codegraph diffBranch blast radius and changed-but-untested.
  • codegraph renameElement-precise rename actuator, dry-run, --apply.actuator
Get running

Three commands. Zero project-specific setup.

Install the CLI, wire the project, and build the graph. The resolved build needs the host project's pub get to be complete; if the package config is missing, the build falls back to syntax-only and tells you.

1

Install the CLI

Global, once per machine. Re-run the same command any time to update.

$dart pub global activate -sgit https://github.com/ArinFaraj/codegraph
2

Wire the project

Stamps a CLAUDE.md block, a freshness hook, and an agent skill. Idempotent, safe to re-run.

$codegraph init --ci
3

Build the graph

Generates the graph JSON (untracked; the hook and CI rebuild it) and per-area maps. The resolved build uses the analyzer element model by default and falls back to syntax-only when the host package config is missing.

$codegraph build
Honest setup, two modes. For a resolved build, run dart pub get in the host project first. If .dart_tool/package_config.json is missing, codegraph build falls back to syntax-only and tells you. The syntax fallback is still zero-setup and never breaks on unresolvable dependencies; it is just less precise.
