The problem

Grep finds text. It doesn't know what's related.

"What breaks if I change this provider" is a graph question, not a text-search question. Text-search is all an agent has by default.

The query surface

Every relationship an agent asks about — down to the call chain.

From "where is X" to "who calls this method" to "what actually runs when this fires, and where does it skip." Output is line-budgeted so a query never blows an agent's context window.

brief

Everything about X, one call

A single context card for a provider, file, area, or symbol — the fields the other verbs would take several calls to assemble.

find

Where is X?

Locate a file, provider, or symbol (class, enum, function) by name, across lib/ and every local package.

readers

What uses this?

Every watcher, reader, and listener of a provider. The blast radius before you change it.

callers

Who calls this method?

Every call site of a method or function — exact file:line, across lib/ and the test dirs. AST-accurate, so no comment or string false hits. The refactor-safety answer grep can't give.

callchain

What runs — and where does it skip?

A static call tree from any method, each hop flagged with the control-flow hazards you'd otherwise read the body to find: [guard] early-return, [try], [swallow] empty catch, [async] fire-and-forget.

impact

What breaks if I change this?

Transitive dependents of a provider or file, several hops out, with a count of screens affected. The blast radius, resolved — not grepped.

wiring

What does this depend on?

A file's full wiring in both directions: declares, watches, imports, imported-by, navigates.

impls

Who implements this?

The whole subtype tree of an interface — transitive, not just direct children — each resolved to its declaration site. Includes test/ fakes scanned on demand.

path

How do these connect?

The shortest relationship path between two files, through imports and shared providers.

unused

What's dead?

Providers and files with zero consumers. A candidate list to confirm, not a delete list.

sym

The API without the file

A symbol's signature, doc comment, and members — plus every file that imports it — without opening the source. Resolves methods by name, not just top-level symbols.

skeleton

The outline without the file

A file's declarations with line numbers, so an agent orients before reading the whole thing.

passport

Session orientation for free

A ~450-token project digest a session-start hook injects automatically — counts, entry points, hot providers, no query needed.

attention

What needs a human

Ambiguous providers, dead code, orphan files, and stale notes — a triage list for a reviewer, not a verdict.

init

Wire up an agent

Installs a CLAUDE.md quickstart, a session hook, and a skill into any project, so agents reach for the graph by default.

build / check

Stay fresh

Regenerate on demand, or gate CI on drift with codegraph check.

Get running

Three commands. Zero project-specific setup.

1

Install the CLI

Global, once per machine. Re-run the same command any time to update.

$dart pub global activate -sgit https://github.com/ArinFaraj/codegraph
2

Wire the project

Stamps a CLAUDE.md block, a freshness hook, and an agent skill. Idempotent, safe to re-run.

$codegraph init --ci
3

Build the graph

Generates the graph JSON (untracked; the hook and CI rebuild it) and per-area maps — commit the markdown maps. From here, the session hook injects a ~450-token project passport at the start of every session — free orientation, no query needed.

$codegraph build
Freshness, without the cost. A session-start hook regenerates only when lib/ or packages/*/lib changed: fail-safe, zero cost per edit. CI can gate on drift with codegraph check. Per-edit regeneration was tried and rejected: seconds of latency for marginal freshness.
